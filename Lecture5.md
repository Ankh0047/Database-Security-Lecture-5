1. Plain Text (Энгийн текст)

Plain text гэдэг нь ямар нэгэн хамгаалалт, кодлол хийгдээгүй шууд уншигдах боломжтой мэдээлэл юм.

Жишээ:

Password: 123456
Name: Bat

Онцлог

Хүн болон компьютер шууд уншиж чадна.

Ямар ч нууцлалгүй.

Хэрэв мэдээлэл алдагдвал хэн ч уншиж ашиглаж чадна.

Сул тал

Маш аюулгүй биш.

Хакер эсвэл гуравдагч этгээд мэдээллийг шууд ойлгож чадна.

2. Encryption (Шифрлэлт)

Encryption гэдэг нь мэдээллийг тусгай алгоритм болон түлхүүр (key) ашиглан кодолж уншигдах боломжгүй хэлбэрт оруулах арга юм.

Жишээ

Plain text:

HELLO

Encrypted text:

XJ29#K1

Онцлог

Тусгай key ашиглан мэдээллийг кодолдог.

Зөв key байвал дахин тайлж (decryption) уншиж болно.

Жишээ хэрэглээ

HTTPS веб холбоо

Онлайн банк

Мессеж хамгаалалт

Давуу тал

Мэдээлэл дамжуулах үед хамгаалалттай.

Зөв түлхүүргүй бол унших боломжгүй.

Сул тал

Хэрэв key алдагдвал мэдээлэл тайлагдаж болно.

3. Hashing

Hashing гэдэг нь мэдээллийг нэг чиглэлтэй алгоритм ашиглан тогтмол урттай код (hash value) болгон хувиргах арга юм.

Жишээ

Password:

123456

Hash:

8d969eef6ecad3c29a3a629280e686cf

Онцлог

Буцааж тайлах боломжгүй (one-way function).

Нэг ижил мэдээлэл үргэлж ижил hash гаргана.

Жижиг өөрчлөлт ч hash-ийг бүрэн өөрчилнө.

Жишээ хэрэглээ

Password хадгалах

File integrity шалгах

Blockchain

Давуу тал

Мэдээллийг анхны хэлбэрт буцаах боломжгүй.

Password хамгаалалтанд маш тохиромжтой.

Сул тал

Анхны мэдээллийг буцааж авах боломжгүй.

Зарим хуучин алгоритм (MD5, SHA1) эмзэг болсон.

Дүгнэлт

Plain text нь огт хамгаалалтгүй тул хамгийн аюулгүй биш.

Encryption нь мэдээллийг хамгаалдаг боловч key алдагдвал эрсдэлтэй.

Hashing нь буцааж тайлах боломжгүй учир password зэрэг мэдээлэл хамгаалахад хамгийн аюулгүй арга гэж үздэг.
